1. Overview

This walkthrough explains Rotary Position Embeddings (RoPE) — a method for encoding position information in transformers — and demonstrates how a small transformer learns induction heads, which are circuits that copy repeating patterns (like "ab…ab" → predict "ab").

🎯 Key Insight

RoPE allows the transformer to attend based on relative position (how far apart tokens are) rather than absolute position (where they appear in the sequence). This is crucial for induction heads to recognize patterns like "the same token appeared N positions ago."

2. RoPE: Rotary Position Embeddings

The Problem

Standard transformers add position embeddings to token embeddings. But this makes attention depend on absolute position, which is bad for copying patterns — you want "token at position 5 attends to similar token at position 2" to work the same way regardless of where in the sequence this happens.

The Solution

RoPE rotates query and key vectors in 2D planes based on their position. The key property: the dot product between a query at position m and key at position n depends only on (m − n), their relative distance.

The Equations (from Chris's notes)

Rotation Frequencies

$$\\theta_i = \\frac{1}{\\text{base}^{2i/d_{\\text{head}}}} \\quad \\text{for } i = 0, 1, \\ldots, \\frac{d_{\\text{head}}}{2} - 1$$

With base = 10000, this gives frequencies ranging from very fast (local patterns) to very slow (global patterns).

RoPE Rotation

For a vector x at position m, and for each pair of dimensions (2i, 2i+1):

$$\\text{RoPE}(x, m)_{2i} = x_{2i} \\cos(m \\theta_i) - x_{2i+1} \\sin(m \\theta_i)$$

$$\\text{RoPE}(x, m)_{2i+1} = x_{2i} \\sin(m \\theta_i) + x_{2i+1} \\cos(m \\theta_i)$$

Attention Score

$$\\text{score}(q_m, k_n) = \\text{RoPE}(q, m) \\cdot \\text{RoPE}(k, n) = f(q, k, m - n)$$

The crucial result: the score depends only on the relative position (m − n), not absolute positions.

💡 Why This Works for Induction

If "abc" appeared at positions 0-2 and again at positions 5-7, RoPE allows the model to recognize that position 5 relates to position 0 in the same way position 6 relates to position 1 — same relative offset, same attention pattern.

3. Induction Heads

What Are Induction Heads?

Induction heads are attention heads that implement the algorithm: "Look for previous instances of the current token, then predict what came next in those previous instances."

Example: [abc][abc] Pattern

Given input: a b c a b c

When predicting the token after the second "a" (position 3):

  1. The induction head attends from position 3 to position 0 (both are "a")
  2. This retrieves information: "after 'a' at position 0 came 'b'"
  3. The model predicts "b" as the next token
Attention Pattern for Induction

For an induction head, the attention matrix shows:

  • Previous token component: Attends to position (i − 1) — standard "look back" behavior
  • Induction component: When the current token appeared before, attend to what followed that previous instance

This creates off-diagonal stripes in the attention heatmap.

The Two-Layer Structure

Layer 0 (Previous Token Heads)

Attend to the previous position. This allows the model to know "what token came before this one" everywhere in the sequence.

Layer 1 (Induction Heads)

Use the information from Layer 0 to find where the current token appeared before, and copy the token that followed those previous instances.

4. Reverse Engineering the Toy Model

What We Found

By analyzing the learned weights of our toy 2-layer RoPE transformer, we can verify it actually learned induction heads:

W_Q and W_K Analysis

The query and key weight matrices show structure that enables position-relative matching. The attention patterns reveal:

  • Head 0 in Layer 0: Previous token head (attends to position i−1)
  • Head 2 in Layer 1: Induction head (attends to matching tokens and copies next token)

Verifying the Circuit

We can verify the induction circuit by:

  1. Feeding the model [abc][abc] patterns
  2. Checking that Layer 0 outputs contain "previous token" information
  3. Verifying Layer 1 induction heads attend correctly
  4. Confirming predictions match the induction pattern
🔍 Reverse Engineering Insight

The W_Q and W_K matrices decompose into components that correspond to: (1) matching query tokens to key tokens (content matching), and (2) RoPE providing position-relative information. The combination enables the full induction algorithm.
